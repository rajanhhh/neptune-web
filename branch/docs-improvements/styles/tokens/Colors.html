Styles

    Colors

    Semantic colours as custom properties for you to use in your application.

    Note: If this is your first time using custom properties in your application, read our guide first to get set up.

    Semantic colors

    Neptune uses semantic colors, and distributes them as custom properties.

    Basic colors

    --color-primary
    --color-secondary
    --color-accent
    --color-positive
    --color-negative
    --color-warning

    Text colors

    --color-text-primary
    --color-text-secondary
    --color-text-inactive
    --color-text-control
    --color-text-important
    --color-text-link
    --color-text-link-hover
    --color-text-link-active
    --color-text-positive
    --color-text-negative
    --color-text-warning

    Control colors

    Control tokens should be used to paint interactive areas in components, such as buttons, the background of a checkbox, or a switch.

    --color-control-accent
    --color-control-accent-hover
    --color-control-accent-active
    --color-control-positive
    --color-control-positive-hover
    --color-control-positive-active
    --color-control-negative
    --color-control-negative-hover
    --color-control-negative-active
    --color-control-warning
    --color-control-warning-hover
    --color-control-warning-active

    Border colors

    We use border tokens for component borders and individual lines.

    --color-border-neutral
    --color-border-focus
    --color-border-positive
    --color-border-negative
    --color-border-warning

    Background colors

    --color-background-screen
    --color-background-elevated
    --color-background-accent
    --color-background-positive
    --color-background-negative
    --color-background-warning
